Chapter 3: Functions

Defining a function:
- var square = function(X) { - return x * x; - }; - console.log(square(12));
Breakdown:
- keyword function - parameters in parentheses - body which has statements that are executed when the function is called. It must always be wrapped in curly braces. - They either produce values or side effects - Return determines the value returned, without an expression after it will be undefined - Parameters are local to the function - Var keyword creates them - If you want to change a global variable that was declared outside the function, you simply do x = new value in the function instead of var x = new value. - Functions can be created within other functions - i.e. nested scope - Each local scope can also see the local scopes that contain it. - Lexical scoping - functions are the only thing that create a new scope - Function variable = function value - var fun = function() {}; - A variable that holds a function can be changed and assigned a new value later - Function Declaration notation: - function square(X) { - return x *X; - } - Not a part of control flow, can be used by all code in the scope of bottom to top flow
Call stack
- The place where the computer stores the context where every time a function is called, the current context is put on top of this stack. - If you pass too many arguments to a function, it ignores the others - So you can pass optional arguments.
