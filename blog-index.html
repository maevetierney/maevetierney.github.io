Looping in Ruby vs. Looping JavaScript Looping in JS and in Ruby aren't that different. The basic ideas of looping using awesome keywords like IF, ELSE, and ELSIF are still in used in Javascript. There's something to be said though. Ruby is really hard to learn. But JS is SUPER hard to learn. Lol. So far! One of the biggest differences in the way JS loops is that many of the formats of the loops have a specific initizlize and condition aspect that have to be included. If you think about it it makes sense, because it's helpful for the program to know thses things while it loops over the data you give it. Ruby just assumes a lot more, until you tell it to not assume it. JS just wants you to tell it everything or it won't assume much at all. There is also this neat thing called break, which is a statement that ends a loop according to the statement.
